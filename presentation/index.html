Blockchain Basics

What is Blockchain in real life

Hi!

In this lecture, we will learn what Blockchain is in real life.

We will follow a real life analogy to better understand the blockchain, as well as why it is different from a traditional database and why it is considered such a unique and powerful technology that people in general appreciate it so much.

What is it? What is blockchain?

  • Data structure
  • Arrangement of data in computer memory
  • Real-life examples:
    • PDF Docs
    • Excel sheets
    • MySQL Table
So, what is it: a data structure or a network?
The answer is: it is a data structure.
It's actually an arrangement that stores information in some way.

So a data structure is nothing but an arrangement of data right in the computers memory, and in real life it can be anything, it can just be pdf files, you can also call a data structure: an excel sheet, a mysql table or any database.

So blockchain is also a data structure, but a different type of data structure.

Imagine it as a book where each page reference to the previouse page through a page number one less than a current page number.

  • Easy to detect if a page has been removed.
  • Easy to arrange the pages & identify suspicious activity. That's why page numbers are important in any agreements.
The real life analogy I was talking about is a book. When you read a book, it is nothing but a kind of blockchain, so you can compare real life blockchain with a book.

Let me show you an example.

Imagine you have a book and each page actually links to an existing page, so the number on each page is the number that actually links to the previous page. This means that we can always organize pages very easily, such is the nature of pages right in a book. Usually we also store the page number in agreements, we actually put the page number in order to determine the sequence.

It turns out that you simply represent one page as one block. But what does such a connection between pages give us?

It makes it very easy to identify a page and it's very easy to put them in a certain order, and it's also very easy to tell if a certain page has been removed from the whole sequence. This is especially important in agreements, because changing just one page of an agreement can completely change its meaning.

Wait! What if the content of the page has been altered?

  • In that case, lets produce the page number by crunching the content of the previous page.
  • Now it makes sense. If anyone changes the content of the page, then the next page number will not match if we try to generate it using altered page content.
Book ordering Block ordering

Implicit that the page builds on the page whose number is one less. Page 5 builds on page 4 (5 minus 1). 84n855, n58uf0, 90fn5n, 8n6d7j represent fingerprints or hashes of the blocks.
But wait, what if the content of the page was changed without changing the page number. Then who would be responsible and how would you determine that?

Okay, in that case let's create a page number using the content of the page, not just the previous page number, so that each page will actually also refer to the content of the previous page.

Imagine lines on a page. Each line on a page, you can think of it as a transaction on the blockchain.

A page has 25 lines or 30 lines, so the next page number will be created based on the content of the previous page. That's exactly what happens in blockchain, and that's how they form the chain, because the content of the next block actually has the signature of the previous block as well.

Let me give you a simple example.

Let's say there is page number four and there is page number five. Page four will have ten lines, in which case page five will also have a summary of the contents of page four.

Now imagine that there are 100 of these pages that are linked together in this way. Is it possible to change the content of page number 26 without affecting the entire chain?

The answer is no, impossible.

You can't make a change so that it can't be detected on the 100th page. Because page 100 contains a summary of page 99, and page 98 is also linked to page 97 and so on.

So if you change the content of page 26 by even one single letter on one line, the whole book is invalid, and it's very easy to detect that somebody has changed the content. That is the concept of blockchain technology.

More about Blockchain

  • In real Blockchain case, each block is built on top of the recent block and use its previous block's content as a signature.
  • Building a block & adding it in the Blockchain is the task of the miner nodes.
  • In public Blockchain it is made computationally difficult to add a block to prevent attacks.
  • Miners try to guess a number in such a way that if it gets crunched with the most recent block's fingerprint than it will create a new fingerprint which will be less that the last/most recent block in Blockchain.
  • It takes time & computational power to add a Block in the Blockchain. Hence there is some reward (currently 6.25 BTC in case of Bitcoin Blockchain).
But there can be certain disadvantages. As I said before, each block is built on top of the previous block, like in a book. In our example one page is built on top of another page so it is very easy to detect any changes and any suspicious activity.

In the real case of the blockchain, it uses the content of the previous blocks as a signature. You know that there is an algorithm in computer science that converts any amount of data of any size into a string of fixed length called a hash. So, in the case of blockchain, what we call a signature is a hash.

So, creating a block and adding it to the blockchain is the task of secondary nodes, but not any general node can add a block.

In a public blockchain, it is computationally difficult to add a block to prevent hacking. In the bitcoin network, for example, it is computationally very expensive and very hard to find a new block.

You might think: Why would anyone make mining very difficult, what's the reason? The reason is simple. What if a hacker changes page number 26 to the right and also changes page number 27 28 29 30 and so on up to page 100 and says, "Okay, there is a new set of pages, please use it as the original"?

If we make it hard for hackers-then they can't invest and create that many blocks very easily because it would cost them a lot, so we reduce the chances of fake pages in our book.

Of course, if you make adding blocks difficult, why would anyone invest money in their computer or servers to create new blocks? The answer is that a miner will be well rewarded for creating blocks.

Private blockchain can choose other methods of block creation. For example, you can sign a contract with the miners and say that only you guys. are allowed to create new blocks and the rest of us are not allowed, this is called private blockchain.

Summary

  • It is a decentralized ledge (data structure) where data is being stored in blocks in the form of transactions.
  • Remove the dependency on the trusted third party for recording the data in Blocks.
  • In public Blockchain more complex algorithm require to avoid the malicious activities.
  • Since each block is built on top of previous Block immutability has been achieved.
  • This all exist in memory of the computers.
  • Every participant of the Blockchain contains the almost same copy of the Blockchain.
That's the idea.

So basically it's a decentralized system. This means that this whole chain of hundreds of blocks, will actually be available to everyone on the network, and every node will contain a copy of this book and can very easily check if anything has been changed.

Public blockchain has more complex algorithms needed to prevent malicious actions.

And because each block is built on top of the previous block, immutability has been achieved, which is why some people say that the data stored in the blockchain is immutable.

So that's the idea behind blockchain.
